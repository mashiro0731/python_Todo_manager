잘한 점:
이번 프로젝트에서 가장 잘했다고 생각하는 부분은 객체지향 프로그래밍 개념을 실제 프로그램에 적용한 것이다.
클래스들로 기능을 분리하여 역할을 명확하게 나누었고, Recurring Task 클래스가 부모 클래스인 Task 클래스로부터
상속 받도록 구현하여 코드의 재사용성을 높였다.

아쉬운 점:
테스트 코드 작성 과정에서 헤멨다. 정상 동작뿐만 아니라 어떤 상황을 엣지 케이스로 고려해야 하는지 판단하는 데 어려움이 있었고,
import 경로 오류 등 여러 문제도 발생하였다. 하지만 이러한 시행착오를 해결하는 과정에서 테스트 작성 방법과 Python 패키지
구조에 대한 이해를 높일 수 있었다.

개선하고 싶은 점:
향후에는 Tkinter와 같은 GUI 라이브러리를 활용하여 사용자가 더욱 편리하게 사용할 수 있는 인터페이스를 추가하고 싶다.

AI 도구 활용 내역:
ChatGPT는 프로젝트 구조 설계, pytest 테스트 코드 작성 방법, 문서화(docstring) 작성 예시, pycodestyle 오류 해결 방법 등에 대한 설명을 제공받는 용도로 활용하였다.
